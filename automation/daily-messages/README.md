# 📅 n8n Daily Telegram Messages

Автоматическая отправка сообщений в Telegram группу по расписанию через n8n.

## 🎯 Что делает:

- ✅ Отправляет сообщение в Telegram группу каждый день в заданное время
- ✅ Настраиваемое расписание (cron expression)
- ✅ Поддержка форматирования текста (HTML/Markdown)
- ✅ Динамическое содержимое (дата, время, переменные)

---

## 🚀 Быстрая установка:

### Шаг 1: Подготовка Telegram

1. **Создайте бота через [@BotFather](https://t.me/BotFather)**
   ```
   /newbot
   Имя бота: Daily Messages Bot
   Username: your_daily_bot
   ```
   Сохраните токен: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`

2. **Получите ID группы**
   - Добавьте [@username_to_id_bot](https://t.me/username_to_id_bot) в группу
   - Отправьте любое сообщение
   - Бот покажет ID группы (например: `-1001234567890`)
   - Удалите бота из группы

3. **Добавьте вашего бота в группу**
   - Добавьте бота в группу
   - Дайте права на отправку сообщений

---

### Шаг 2: Настройка n8n

#### Вариант A: Импорт готового workflow

1. Откройте n8n
2. Нажмите **"Import from File"** или **"Import from URL"**
3. Выберите файл `daily_telegram_message.json`
4. Workflow импортирован!

#### Вариант B: Создание с нуля (5 минут)

1. **Создайте новый Workflow** в n8n
2. **Добавьте узел "Schedule Trigger"**
   - Перетащите из панели слева
   - Настройте расписание (см. ниже)
3. **Добавьте узел "Telegram"**
   - Перетащите из панели
   - Выберите "Send Message"
   - Подключите credentials
4. **Соедините узлы**
   - Schedule Trigger → Telegram
5. **Активируйте workflow**

---

### Шаг 3: Настройка Telegram Credentials в n8n

1. В n8n откройте **Settings → Credentials**
2. Нажмите **"New Credential"**
3. Выберите **"Telegram API"**
4. Заполните:
   ```
   Name: Telegram Bot
   Access Token: 1234567890:ABCdefGHIjklMNOpqrsTUVwxyz
   ```
5. Нажмите **"Save"**

---

### Шаг 4: Настройка узлов

#### Schedule Trigger (расписание)

**Примеры cron выражений:**

```bash
# Каждый день в 9:00
0 9 * * *

# Каждый день в 18:00
0 18 * * *

# Каждый день в 9:00 и 18:00
0 9,18 * * *

# Понедельник-Пятница в 9:00
0 9 * * 1-5

# Каждый час
0 * * * *

# Каждые 30 минут
*/30 * * * *
```

**Формат:** `минута час день месяц день_недели`

**Настройка в n8n:**
1. Откройте узел "Schedule Trigger"
2. Mode: "Custom"
3. Cron Expression: `0 9 * * *` (каждый день в 9:00)
4. Timezone: `Europe/Moscow` (ваш часовой пояс)

#### Telegram Node (отправка сообщения)

**Базовая настройка:**

```
Chat ID: -1001234567890    # ← ID вашей группы
Text: Ваше сообщение       # ← Текст сообщения
```

**Примеры сообщений:**

```html
<!-- Простое сообщение -->
🌅 Доброе утро!
Это ежедневное напоминание.

<!-- С датой и временем -->
📅 Сегодня: {{ $now.format('dd MMMM yyyy') }}
⏰ Время: {{ $now.format('HH:mm') }}

<!-- HTML форматирование -->
<b>Важное объявление!</b>

<i>Сегодняшние задачи:</i>
• Задача 1
• Задача 2
• Задача 3

<code>Статус:</code> Всё работает ✅

<!-- Markdown форматирование -->
*Важное объявление!*

_Сегодняшние задачи:_
• Задача 1
• Задача 2
• Задача 3

`Статус:` Всё работает ✅
```

**Дополнительные параметры:**

- **Parse Mode:** `HTML` или `Markdown` (для форматирования)
- **Disable Notification:** `true/false` (тихая отправка)
- **Disable Web Page Preview:** `true/false` (скрыть превью ссылок)

## 📝 Шаблон workflow для копирования:

### Простейший вариант (2 узла):

```
1. Schedule Trigger (0 9 * * *)
   └──> 2. Telegram (Send Message)
```

### С условием (день недели):

```
1. Schedule Trigger (0 9 * * *)
   └──> 2. IF (проверка дня недели)
         ├──> true: 3. Telegram (сообщение для будней)
         └──> false: 4. Telegram (сообщение для выходных)
```

### С переменными (динамический текст):

```
1. Schedule Trigger (0 9 * * *)
   └──> 2. Set (установить переменные)
         └──> 3. Telegram (использовать переменные)
```

## 🔧 Расширенные возможности:

### Динамические данные из API

Добавьте узел HTTP Request для получения данных:

```
1. Schedule Trigger
   └──> 2. HTTP Request (получить погоду)
         └──> 3. Telegram (отправить с погодой)
```

**Пример текста:**
```
🌤 Доброе утро!

Погода сегодня:
🌡 {{ $json.temp }}°C
💧 Влажность: {{ $json.humidity }}%

Хорошего дня!
```

### Отправка в несколько групп

Используйте узел "Split In Batches":

```
1. Schedule Trigger
   └──> 2. Set (список групп)
         └──> 3. Split In Batches
               └──> 4. Telegram (отправить в каждую)
```

### Чтение сообщения из Google Sheets

```
1. Schedule Trigger
   └──> 2. Google Sheets (прочитать сообщение)
         └──> 3. Telegram (отправить)
```

## ⚙️ Настройка cron расписания:

### Формат cron:
```
* * * * *
│ │ │ │ │
│ │ │ │ └─── День недели (0-7, 0 и 7 = Воскресенье)
│ │ │ └───── Месяц (1-12)
│ │ └─────── День месяца (1-31)
│ └───────── Час (0-23)
└─────────── Минута (0-59)
```

### Примеры:

| Cron Expression | Описание |
|-----------------|----------|
| `0 9 * * *` | Каждый день в 9:00 |
| `30 18 * * *` | Каждый день в 18:30 |
| `0 9,18 * * *` | Каждый день в 9:00 и 18:00 |
| `0 9 * * 1-5` | Будни в 9:00 |
| `0 9 * * 6,0` | Выходные в 9:00 |
| `0 9 1 * *` | 1-е число каждого месяца в 9:00 |
| `0 9 * * 1` | Каждый понедельник в 9:00 |
| `*/30 * * * *` | Каждые 30 минут |
| `0 */2 * * *` | Каждые 2 часа |
| `0 9-18 * * 1-5` | Будни, каждый час с 9 до 18 |

### Полезные ссылки:

- [Crontab Guru](https://crontab.guru/) - генератор cron выражений
- [Crontab Generator](https://crontab-generator.org/) - визуальный редактор

## 🆘 Решение проблем:

### Сообщение не отправляется

**Проблема:** Ошибка `Bad Request: chat not found`

**Решение:**
1. Проверьте Chat ID (должен начинаться с `-100`)
2. Убедитесь что бот добавлен в группу
3. Отправьте любое сообщение в группу (для активации)

---

**Проблема:** Ошибка `Unauthorized`

**Решение:**
1. Проверьте токен бота
2. Пересоздайте credentials в n8n
3. Проверьте что токен не содержит лишних пробелов

---

**Проблема:** Сообщение не форматируется

**Решение:**
1. Проверьте Parse Mode (HTML или Markdown)
2. Проверьте синтаксис форматирования
3. Попробуйте без форматирования

---

### Workflow не запускается по расписанию

**Проблема:** Workflow неактивен

**Решение:**
1. Проверьте что workflow **Active** (зелёный переключатель)
2. Проверьте executions в n8n (есть ли ошибки)

---

**Проблема:** Неправильное время

**Решение:**
1. Проверьте timezone в Schedule Trigger
2. Проверьте cron expression ([crontab.guru](https://crontab.guru/))
