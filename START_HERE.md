## 🤖 Доступные агенты

### Команда разработки:

1. **@teamlead** - Координатор команды, разбивает задачи
2. **@architect** - Проектирование архитектуры  
3. **@backend** - Python, API, БД, Telegram боты
4. **@frontend** - React, UI, JavaScript
5. **@devops** - Деплой, CI/CD, инфраструктура
6. **@qa** - Тесты, проверка качества
7. **@docs** - Документация, README
8. **@research** - Исследование решений, библиотек
9. **@refactor** - Оптимизация, рефакторинг

### Автоматическая активация:

Агенты автоматически активируются при работе с файлами:
- `.py` файлы → **Backend Agent**
- `.tsx`, `.jsx` файлы → **Frontend Agent**
- `.md` файлы → **Documentation Agent**

```
2. @architect → Спроектировать систему уведомлений
3. @backend → Реализовать отправку email
4. @frontend → Добавить настройки уведомлений
5. @qa → Написать тесты
6. @docs → Документировать API
```

## 🎯 Workflow примеры

### Новая фича

```mermaid
@teamlead → Plan → @research → @architect → @backend/@frontend → @qa → @docs
```

1. **TeamLead** разбивает на задачи
2. **Research** исследует решения
3. **Architect** проектирует архитектуру
4. **Backend/Frontend** реализуют
5. **QA** тестирует
6. **Docs** документирует

### Баг фикс

```mermaid
@backend → Analyze → Fix → @qa → Test → @devops → Deploy
```

1. **Backend** анализирует и исправляет
2. **QA** добавляет тест для edge case
3. **DevOps** деплоит на production

## 🤝 Помощь и поддержка

### Проблемы с агентами?

1. **Агент не активируется**
   - Проверьте что файл правила существует
   - Проверьте frontmatter (YAML в начале файла)

2. **Агент делает не то**
   - Уточните задачу более детально
   - Дайте больше контекста

3. **Нужен новый агент**
   - Создайте новое правило в `.cursor/rules/agents/`
   - Используйте существующие как шаблон

### Где искать информацию:

1. **Быстрые примеры** → `.cursor/AGENTS_QUICKSTART.md`
2. **Детали агентов** → `.cursor/AGENTS.md`
3. **Полный отчет** → `AGENTS_SYSTEM_SUMMARY.md`
