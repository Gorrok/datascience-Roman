# 🤖 Telegram Member Tracker Bot

Бот для отслеживания вступлений и выходов участников в Telegram каналах с сохранением информации в базу данных.

## 🚀 Функционал

- **Автоматическое отслеживание** вступлений/выходов участников каналов
- **Сохранение данных** о пользователях (ID, username, имя, фамилия)
- **Логирование активности** всех изменений статуса участников
- **Статистика** по каналам
- **Поддержка нескольких каналов** одновременно

## 📋 Требования

- Python 3.8+
- Telegram Bot Token (получить у [@BotFather](https://t.me/botfather))
- Google Cloud Platform аккаунт с включенным Google Sheets API
- Права администратора в каналах для отслеживания
- Google Sheets таблица для хранения данных

## ⚙️ Установка

1. **Клонируйте репозиторий:**
   ```bash
   git clone <repository-url>
   cd telegram_member_tracker
   ```

2. **Установите зависимости:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Настройте переменные окружения:**
   ```bash
   cp config_example.txt .env
   # Отредактируйте .env файл
   ```

4. **Заполните .env файл:**
   ```env
   TELEGRAM_BOT_TOKEN=ваш_токен_бота_здесь
   MONITORED_CHANNELS=@channel1,@channel2
   DATABASE_URL=sqlite:///member_tracker.db
   LOG_LEVEL=INFO
   LOG_FILE=logs/bot.log
   ```

## 🔧 Настройка бота

### 1. Создание бота в Telegram

1. Перейдите к [@BotFather](https://t.me/botfather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Скопируйте токен и вставьте в `.env`

### 2. Добавление бота в каналы

1. Добавьте бота как администратора в каждый канал
2. Убедитесь, что бот имеет права на чтение сообщений и управление участниками

### 3. Настройка Google Sheets API

#### Создание сервисного аккаунта:

1. Перейдите в [Google Cloud Console](https://console.cloud.google.com/)
2. Создайте новый проект или выберите существующий
3. Включите Google Sheets API в разделе "APIs & Services"
4. Перейдите в "Credentials" → "Create credentials" → "Service account"
5. Укажите имя сервисного аккаунта и создайте его
6. Перейдите в созданный сервисный аккаунт → "Keys" → "Add key" → "JSON"
7. Скачайте JSON файл и переименуйте его в `credentials.json`
8. Поместите `credentials.json` в папку с ботом

#### Настройка Google Sheets таблицы:

1. Создайте новую Google Sheets таблицу
2. Скопируйте ID таблицы из URL (часть между `/d/` и `/edit`)
3. Предоставьте доступ к таблице сервисному аккаунту (email из JSON файла) с правами "Editor"
4. В таблице автоматически создадутся листы "members" и "activity"

### 4. Указание каналов для мониторинга

В файле `.env` укажите usernames каналов через запятую:
```env
MONITORED_CHANNELS=@my_channel,@another_channel,@third_channel
```

## 📊 Использование

### Команды бота:
- `/start` - запуск и информация о боте
- `/stats` - статистика по каналам
- `/help` - справка

### Просмотр данных:

Google Sheets таблица содержит два листа:

1. **`members`** - информация об участниках
   - User ID - уникальный идентификатор пользователя
   - Username - username пользователя (@username)
   - First Name - имя пользователя
   - Last Name - фамилия пользователя
   - Channel Username - username канала
   - Joined At - дата и время вступления
   - Left At - дата и время выхода (если вышел)
   - Is Active - активен ли участник (true/false)
   - Status - статус в канале (member/administrator/creator)
   - Notes - дополнительные заметки

2. **`activity`** - логи активности
   - User ID - идентификатор пользователя
   - Channel Username - username канала
   - Activity Type - тип активности (joined/left/promoted/demoted)
   - Old Status - предыдущий статус
   - New Status - новый статус
   - Timestamp - дата и время события
   - Details - дополнительные детали

## 🔍 Отслеживаемые события

Бот автоматически отслеживает следующие изменения статуса участников:

- **joined** - новый участник вступил в канал
- **left** - участник вышел/был удален из канала
- **promoted** - участник повышен до администратора
- **demoted** - администратор понижен до обычного участника
- **status_changed** - другие изменения статуса

## 🛠️ Разработка

### Добавление нового функционала:

1. Модифицируйте модели в `models.py`
2. Добавьте обработчики в `bot.py`
3. Обновите базу данных:
   ```python
   # В Python shell
   from models import Base, engine
   Base.metadata.create_all(engine)
   ```

### Логирование:

Бот использует loguru для структурированного логирования. Логи сохраняются в файл и выводятся в консоль.

## 🚨 Устранение неполадок

### Бот не видит изменения в канале:
- Убедитесь, что бот добавлен как администратор
- Проверьте права бота в канале
- Проверьте корректность username канала в `.env`

### Ошибка подключения к базе данных:
- Проверьте путь к файлу БД
- Убедитесь, что директория доступна для записи

### Проблемы с зависимостями:
```bash
pip install --upgrade pip
pip install -r requirements.txt --force-reinstall
```
