## Примеры документации

### Гайд по узлу:
```markdown
# HTTP Request Node

## Описание
Узел для выполнения HTTP запросов к внешним API.

## Параметры
- **URL**: Адрес для запроса
- **Method**: HTTP метод (GET, POST, PUT, DELETE)
- **Headers**: Заголовки запроса
- **Body**: Тело запроса

## Примеры

### GET запрос:
```json
{
  "url": "https://api.example.com/users",
  "method": "GET",
  "headers": {
    "Authorization": "Bearer {{$credentials.apiKey}}"
  }
}
```

### POST запрос:
```json
{
  "url": "https://api.example.com/users",
  "method": "POST",
  "headers": {
    "Content-Type": "application/json"
  },
  "body": {
    "name": "{{$json.name}}",
    "email": "{{$json.email}}"
  }
}
```

## Частые ошибки
- **401 Unauthorized**: Проверьте API ключ
- **404 Not Found**: Проверьте URL
- **Timeout**: Увеличьте таймаут
```

### Troubleshooting гайд:
```markdown
# Решение проблем с n8n

## Проблема: Workflow не запускается

### Симптомы:
- Workflow не выполняется
- Ошибка в логах

### Возможные причины:
1. Неправильная настройка триггера
2. Ошибки в узлах
3. Проблемы с credentials

### Решение:
1. Проверьте настройки триггера
2. Протестируйте каждый узел отдельно
3. Проверьте credentials

### Профилактика:
- Тестируйте workflow перед запуском
- Используйте логирование
- Настройте мониторинг
```
