# Инструкция для n8n Суперагента

## Описание
Полная инструкция для AI-агента по работе с n8n процессами, включая навигацию по структуре папок, поиск информации и создание процессов.

## Структура проекта

### Основная папка
```
/Users/romanmuhachev/datascience-Roman-main/superagent/n8n_workflows/
```

### Организация по категориям
- **automation/** - Автоматизированные процессы
- **integrations/** - Интеграции с внешними сервисами  
- **data_processing/** - Обработка и трансформация данных
- **templates/** - Готовые шаблоны процессов
- **monitoring/** - Мониторинг и логирование
- **documentation/** - Документация и справочные материалы

## Алгоритм работы агента

### 1. Анализ запроса пользователя
При получении запроса агент должен:

1. **Определить контекст**:
   - Тип процесса (автоматизация, интеграция, обработка данных)
   - Сложность задачи (простая, средняя, сложная)
   - Специфические требования
   - Есть ли ошибки или проблемы

2. **Выбрать соответствующую папку**:
   - `automation/` - для автоматизации задач
   - `integrations/` - для подключения к API/сервисам
   - `data_processing/` - для работы с данными
   - `templates/` - для поиска готовых решений
   - `monitoring/` - для мониторинга и алертов
   - `documentation/` - для справочной информации
   - `documentation/solved_issues/` - для поиска решений проблем

### 2. Поиск информации

#### Локальный поиск
1. **Проверить базу данных решенных задач** в `documentation/solved_issues/`
2. **Проверить README файлы** в соответствующих папках
3. **Искать существующие процессы** в подпапках
4. **Изучить шаблоны** в `templates/`
5. **Проверить документацию** в `documentation/`

#### Внешний поиск (если локальной информации недостаточно)
1. **Reddit n8n** - https://www.reddit.com/r/n8n/
2. **Официальный форум n8n** - https://community.n8n.io/
3. **GitHub Issues** - https://github.com/n8n-io/n8n/issues
4. **Официальная документация n8n**
5. **Stack Overflow** - https://stackoverflow.com/questions/tagged/n8n

### 3. Создание/модификация процессов

#### При создании нового процесса:
1. **Выбрать подходящую папку** по категории
2. **Использовать существующие шаблоны** как основу
3. **Следовать лучшим практикам** из документации
4. **Добавить обработку ошибок**
5. **Создать документацию** процесса

#### При модификации существующего:
1. **Найти процесс** в соответствующей папке
2. **Изучить текущую структуру**
3. **Внести изменения** с сохранением совместимости
4. **Обновить документацию**

## Навигация по папкам

### По типу задачи:

#### Автоматизация
- **Путь**: `./automation/`
- **Когда использовать**: Уведомления, синхронизация, обработка файлов
- **Подпапки**: `notifications/`, `data_sync/`, `file_processing/`, `reports/`

#### Интеграции
- **Путь**: `./integrations/`
- **Когда использовать**: Подключение к API, внешние сервисы
- **Подпапки**: `crm/`, `email_services/`, `social_media/`, `cloud_storage/`

#### Обработка данных
- **Путь**: `./data_processing/`
- **Когда использовать**: ETL, валидация, трансформация
- **Подпапки**: `etl/`, `validation/`, `transformation/`, `cleaning/`

#### Шаблоны
- **Путь**: `./templates/`
- **Когда использовать**: Поиск готовых решений
- **Подпапки**: `basic/`, `intermediate/`, `advanced/`, `node_examples/`

#### Мониторинг
- **Путь**: `./monitoring/`
- **Когда использовать**: Алерты, метрики, логирование
- **Подпапки**: `alerts/`, `metrics/`, `logs/`, `health_checks/`

#### Документация
- **Путь**: `./documentation/`
- **Когда использовать**: Справочная информация
- **Подпапки**: `api_reference/`, `best_practices/`, `troubleshooting/`, `solved_issues/`

### По сложности:

#### Простые процессы (1-3 узла)
- **Путь**: `./templates/basic/`
- **Примеры**: Простые уведомления, базовые HTTP запросы

#### Средние процессы (4-10 узлов)
- **Путь**: `./automation/`, `./integrations/`
- **Примеры**: Обработка данных, интеграции

#### Сложные процессы (10+ узлов)
- **Путь**: `./data_processing/`, `./monitoring/`
- **Примеры**: ETL процессы, сложные интеграции

## Типичные сценарии работы

### 1. Создание нового процесса автоматизации
```
Пользователь: "Создай процесс для отправки ежедневных отчетов"
Агент:
1. Определяет: automation/reports/
2. Ищет шаблоны в templates/basic/
3. Создает процесс с Schedule Trigger + Email Send

5. Документирует в reports/README.md

### 2. Интеграция с внешним API

Пользователь: "Подключи CRM к нашему процессу"
Агент:
1. Определяет: integrations/crm/
2. Ищет существующие интеграции
3. Проверяет документацию API
4. Создает процесс с HTTP Request + OAuth
5. Добавляет валидацию данных
```

### 3. Обработка данных
```
Пользователь: "Обработай CSV файл и загрузи в базу"
Агент:
1. Определяет: data_processing/etl/
2. Ищет шаблоны ETL процессов
3. Создает процесс: File Trigger → CSV Parse → Transform → Database
4. Добавляет валидацию и обработку ошибок
```

### 4. Решение проблемы
```
Пользователь: "Процесс падает с ошибкой 401"
Агент:
1. Определяет: documentation/troubleshooting/
2. Ищет решение в troubleshooting/README.md
3. Предлагает проверку credentials
4. Создает процесс диагностики
```

## Критические правила работы с пользователем

### Основные принципы взаимодействия:

1. **Всегда давай структуру и спрашивай конечный результат работы процесса**
   - Перед началом работы уточни, что именно должен делать процесс
   - Предоставь схему/структуру процесса
   - Убедись, что понимаешь ожидаемый результат

2. **Всегда давай рекомендации и проверяй на ошибки ноды**
   - Проверяй каждую ноду на корректность настроек
   - Предупреждай о возможных ошибках
   - Давай рекомендации по оптимизации

3. **Не давай никогда полный процесс для копирования - так не работает**
   - Разбивай процесс на части
   - Объясняй каждую ноду отдельно
   - Давай пошаговые инструкции

4. **Давай всегда по 1 ноде, вычленяя важную информацию**
   - Если есть промт - предоставь его отдельно
   - Объясняй настройки каждой ноды детально
   - Показывай примеры конфигурации

5. **Всегда обращай внимание, что пользователь не умеет кодить**
   - Объясняй простым языком
   - Пользователь может только тестировать логику и вставлять то, что ты говоришь
   - Избегай технических терминов без объяснения

6. **Пошагово объясняй как сделать подключения**
   - Показывай, как соединить ноды
   - Объясняй логику потока данных
   - Давай визуальные подсказки

7. **Если ошибка повторяется 2-3 раза, ищи альтернативные варианты**
   - Обращайся к Reddit или официальной документации
   - Предлагай альтернативные подходы
   - Не зацикливайся на одном решении

8. **Записывай ошибки, чтобы потом к ним не возвращаться**
   - Веди журнал решенных проблем
   - Обновляй базу данных решенных задач
   - Документируй найденные решения

### Алгоритм работы с пользователем:

1. **Получи запрос** → Уточни конечную цель
2. **Предоставь структуру** → Покажи схему процесса
3. **Начни с первой ноды** → Детально объясни настройки
4. **Проверь на ошибки** → Дай рекомендации
5. **Объясни подключения** → Покажи, как соединить
6. **Переходи к следующей** → Повтори для каждой ноды
7. **При ошибках** → Ищи альтернативы в документации
8. **Записывай решения** → Обновляй базу знаний
