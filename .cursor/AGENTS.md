# 🤖 Команда разработки Cursor Agents

Эта команда агентов работает как полноценная команда разработки. Каждый агент специализируется на своей области и может делегировать задачи другим агентам.

---

## 👥 Структура команды

```
           TeamLead (Координатор)
                    |
    ┌───────────────┼───────────────┐
    |               |               |
Architect      Development        Support
    |               |               |
    |         ┌─────┼─────┐        |
    |      Backend Frontend     ┌──┼──┐
    |         |         |       QA DevOps
    |         |         |        |    |
    └─────────┴─────────┴────────┴────┴──> Documentation
                                           Research
                                           Refactoring
```

## 🎯 Агенты и их роли

### 1️⃣ TeamLead (Главный координатор)
**Команда для вызова**: `@teamlead`  
**Когда использовать**: Когда у вас сложная задача, которую нужно разбить на подзадачи

**Что делает**:
- Анализирует задачу и разбивает ее на подзадачи
- Назначает подзадачи другим агентам
- Координирует работу команды
- Следит за прогрессом
- Объединяет результаты работы

**Правило**: `.cursor/rules/agents/teamlead.mdc`

---

### 2️⃣ Architect (Архитектор)
**Команда для вызова**: `@architect`  
**Когда использовать**: Проектирование новых фич, архитектурные решения

**Что делает**:
- Проектирует архитектуру новых фич
- Выбирает технологии и подходы
- Создает технические планы
- Проводит code review на уровне архитектуры
- Оценивает технический долг

**Правило**: `.cursor/rules/agents/architect.mdc`

---

### 3️⃣ Backend Developer
**Команда для вызова**: `@backend`  
**Когда использовать**: Работа с Python, API, базами данных, Telegram ботами

**Что делает**:
- Разработка API endpoints
- Работа с базами данных (SQL, SQLAlchemy)
- Telegram боты (aiogram, python-telegram-bot)
- Интеграции с внешними API
- Обработка данных

**Правило**: `.cursor/rules/agents/backend.mdc`

---

### 4️⃣ Frontend Developer

## 🚀 Как использовать агентов

### Базовое использование

1. **Простая задача** - обращайтесь напрямую к специализированному агенту:
   ```
   @backend Добавь новый endpoint для получения статистики пользователя
   ```

2. **Сложная задача** - начинайте с TeamLead:
   ```
   @teamlead Мне нужно создать систему уведомлений с email и Telegram
   ```

3. **Неясная задача** - начните с Research:
   ```
   @research Какой лучший способ хранить большие файлы в Telegram боте?
   ```

### Продвинутое использование

#### Цепочка агентов
```
@teamlead Создай фичу для экспорта данных в PDF
  └─> @architect Спроектируй архитектуру
       └─> @backend Реализуй бэкенд
            └─> @frontend Добавь кнопку экспорта
                 └─> @qa Напиши тесты
                      └─> @docs Обнови документацию
```

#### Параллельная работа
```
@teamlead Оптимизируй проект telegram-analytics-saas
  ├─> @refactor Оптимизируй бэкенд
  ├─> @frontend Оптимизируй фронтенд
  └─> @devops Настрой кэширование
```

## 📋 Примеры использования

### Пример 1: Новая фича
```
Пользователь: @teamlead Добавь систему подписок с оплатой через Stripe

TeamLead анализирует:
1. Разбивает задачу на подзадачи
2. Создает план работы

TeamLead делегирует:
@architect → Спроектируй схему БД и API
@research → Изучи Stripe API и лучшие практики
@backend → Реализуй интеграцию с Stripe
@frontend → Создай страницы оплаты
@qa → Напиши тесты для платежей
@docs → Задокументируй процесс оплаты
```

### Пример 2: Баг фикс
```
Пользователь: @backend Бот падает при обработке больших групп

Backend анализирует:
1. Изучает логи
2. Находит проблему (memory overflow)
3. Исправляет (добавляет пагинацию)
4. Просит @qa написать тест для этого случая
```

### Пример 3: Рефакторинг
```
Пользователь: @refactor Оптимизируй проект telegram_member_tracker

Refactoring делает:
1. Анализирует код
2. Находит узкие места
3. Предлагает улучшения
4. Реализует оптимизации
5. Просит @qa проверить что ничего не сломалось
```

## 🎓 Workflow примеры

### Sprint Planning
```bash
# 1. Планирование
@teamlead У меня есть 3 задачи на эту неделю:
- Добавить экспорт в Excel
- Исправить баг с webhook
- Обновить документацию

# TeamLead создаст план и распределит задачи
```

### Code Review
```bash
# 1. Архитектурный ревью
@architect Посмотри на мой новый модуль для работы с платежами

# 2. Детальный ревью
@backend Проверь логику обработки транзакций
@qa Достаточно ли тестов?
```

### Deployment
```bash
# Полный цикл деплоя
@devops Задеплой telegram-analytics-saas на production
  └─> DevOps проверяет:
       - Есть ли тесты? (@qa)
       - Обновлена ли документация? (@docs)
       - Готова ли инфраструктура?
       - Запускает деплой
```

## 🔧 Настройка агентов

Все агенты настраиваются через правила в `.cursor/rules/agents/`:

```
.cursor/rules/agents/
├── teamlead.mdc       # Координация команды
├── architect.mdc      # Архитектура
├── backend.mdc        # Backend разработка
├── frontend.mdc       # Frontend разработка
├── devops.mdc         # DevOps и инфраструктура
├── qa.mdc            # Тестирование
├── documentation.mdc  # Документация
├── research.mdc       # Исследования
└── refactoring.mdc    # Рефакторинг
```

## 💡 Советы по использованию

1. **Начинайте с TeamLead** для сложных задач - он сам разберется кого привлечь
2. **Используйте специализированных агентов** для быстрых точечных задач
3. **Research перед реализацией** - сэкономит время на переделках
4. **Architect для новых фич** - поможет избежать архитектурных проблем
5. **QA после изменений** - предотвратит регрессии
6. **Documentation в конце** - не забывайте документировать
